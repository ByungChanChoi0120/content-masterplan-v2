뉴스레터 구조

[1] 공감 (3줄) — 특정 업종의 아침, "이거 당신 얘기 아닌가요?"

[2] 결과 (1줄 + 숫자) — "여기서는 이렇게 바꿨습니다. 1시간 → 5분."

[3] 실제 화면 (캡처 2~3장) — 뽀짝이가 실제로 한 과정

[4] 왜 되는지 (1줄) — "봇이 일하는 근거 = 정책 문서. 문서 고치면 봇이 바뀜."

[5] 이런 업종에서 특히 효과적 (리스트) — 세무사/프랜차이즈/D2C/병원/법무 등

[6] 질문 (1줄) — "당신 회사에서 가장 반복되는 업무는 뭔가요?" → 답장 유도 = PMF 데이터

P0 / P1 / P2

4/3 office-hours 진단 반영 — 콘텐츠보다 "실사용자 파악 + 트래킹 체계"가 급선무

P0 — 토요일 설치 세션

실사용자 파악 — 이번 주 최우선

  • 참여자 옆에서 직접 설치 과정 관찰 (어디서 막히는지 기록)
  • "설치했어요? 어떻게 쓰고 있어요?" 직접 물어보기
  • 설치 완료한 사람: 업종, 어떤 업무에 쓰려는지 기록
  • 설치 안 한 사람: 왜 안 했는지 (몰라서/필요 없어서/막혀서) 기록
  • 핵심 데이터: 참여자 N명 중 설치 완료 N명, 실사용 N명
  • 1:1 팔로업 연락선 확보 — 카톡 1:1 추가 또는 설치자 전용 소그룹방 개설
  • "다음 주에 한 번 여쭤볼게요" 미리 동의 받기
P1 — 금요일 회의

트래킹 체계 확정

  • "실사용자" 정의 합의 (예: 설치 완료 + 1주 내 1회 이상 사용)
  • 설치 수 트래킹 방법 확정 — 태현 대표와 대시보드/UTM
  • 다혜님 소재 협조 범위
  • 웨비나 참가자 이메일 리스트 확보
  • 뉴스레터 발행 요일 확정
P2 — 콘텐츠 세팅

토요일 결과 나온 후 본격 착수

  • 스티비 계정 + 발신자 인증 + 구독 폼
  • 뉴스레터 템플릿 (텍스트 중심, 이미지 2~3장)
  • 인스타 + 스레드 계정
  • UTM 링크
  • 뽀짝이 실제 업무 5가지별 캡처 + 숫자 정리
  • 카톡방 반복 질문 정리 (업종별 분류)
